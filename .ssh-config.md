## Git SSH Wrapper
Since `openssh-client` is not available in the sandbox, we use a paramiko-based wrapper.

- **Wrapper script**: `~/.ssh/git-ssh-wrapper.py`
- **Dependency**: `paramiko` (installed via pip3)

## Usage
```bash
# Push to GitHub
cd /home/z/Zenic-Agents
GIT_SSH_COMMAND="python3 /home/z/.ssh/git-ssh-wrapper.py" git push origin main

# Pull from GitHub
GIT_SSH_COMMAND="python3 /home/z/.ssh/git-ssh-wrapper.py" git pull origin main

# Set as default for the repo (optional)
cd /home/z/Zenic-Agents
git config core.sshCommand "python3 /home/z/.ssh/git-ssh-wrapper.py"
```

## Permanent Config (recommended)
```bash
cd /home/z/Zenic-Agents
git config core.sshCommand "python3 /home/z/.ssh/git-ssh-wrapper.py"
```
This makes `git push` and `git pull` work without the `GIT_SSH_COMMAND` prefix.
